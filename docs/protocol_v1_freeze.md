# 超声声纳 360 度扫描正式协议 v1.0

## 1. 适用范围

本协议用于上位机与 STM32 下位机之间的正式通信。

适用功能：

- 正式扫描模式
- 调试单点模式
- 调试连续角度模式
- 原始波形上传
- 包络波上传
- 状态上报
- 数据记录与回放

本协议是项目实现基线。后续代码必须以本文件为准。

## 2. 基础约束

- 物理链路：UART
- 推荐波特率：
  - 结果模式：115200
  - 波形模式：460800
- 字节序：小端
- 正式协议：二进制帧
- ASCII：仅用于 bring-up 日志，不承载正式业务

统一量纲：

- 角度：`0.01 degree`
- 距离：`mm`
- 声速：`m/s`
- 时间：`ms`

## 3. 通用帧格式

```text
+----------+---------+---------+---------+---------+-----------+---------+
| SOF(2)   | Ver(1)  | Cmd(1)  | Seq(2)  | Len(2)  | Payload   | CRC16(2)|
+----------+---------+---------+---------+---------+-----------+---------+
```

字段定义：

- `SOF`：固定 `0x55 0xAA`
- `Ver`：当前固定 `0x01`
- `Cmd`：命令字
- `Seq`：帧序号，主机发起时递增，设备应答回同一序号
- `Len`：负载长度
- `Payload`：负载内容
- `CRC16`：对 `Ver + Cmd + Seq + Len + Payload` 做 CRC16-IBM

帧最大长度：

- `Payload <= 1024`
- 单帧总长度不超过 `1034` 字节

## 4. 通用结果码

| 值 | 名称 | 含义 |
|---|---|---|
| `0x00` | OK | 成功 |
| `0x01` | FAIL | 通用失败 |
| `0x02` | BUSY | 设备忙 |
| `0x03` | INVALID_PARAM | 参数非法 |
| `0x04` | INVALID_STATE | 当前状态不允许 |
| `0x05` | CRC_ERROR | 校验错误 |
| `0x06` | UNSUPPORTED | 不支持的命令 |
| `0x07` | TIMEOUT | 动作超时 |
| `0x08` | HARDWARE_ERROR | 硬件错误 |

## 10. 通用 ACK 负载

`0x04 ACK`

```text
+------------+------------+------------+------------------+
| result(1)  | status(1)  | ref_cmd(1) | detail_len(1)    |
+------------+------------+------------+------------------+
| detail(n)                                            |
+------------------------------------------------------+
```

- `ref_cmd`：对应的命令字
- `detail`：按命令不同附加信息

`0x05 NACK`

```text
+------------+------------+------------+--------------+
| result(1)  | status(1)  | ref_cmd(1) | error_code(1)|
+------------+------------+------------+--------------+
```

## 11. GET_DEVICE_INFO `0x01`

请求负载为空。

ACK detail：

```text
+----------------+----------------+----------------+----------------+
| fw_major(1)    | fw_minor(1)    | hw_major(1)    | hw_minor(1)    |
+----------------+----------------+----------------+----------------+
| max_baud(4)    | capabilities(4)                                  |
+---------------------------------------------------------------+
```

`capabilities` 位定义：

- bit0：正式扫描
- bit1：单点调试
- bit2：连续调试扫描
- bit3：原始波形上传
- bit4：包络上传
- bit5：记录索引扩展

## 12. ZEROING `0x02`

请求负载为空。

允许状态：

- IDLE
- READY
- ERROR

ACK detail：

```text
+----------------+----------------+
| zero_done(1)   | current_angle(2)|
+----------------+----------------+
```

## 13. STOP `0x03`

请求负载为空。

允许状态：

- ZEROING
- MOVING
- MEASURING
- SCANNING
- DEBUG_SCANNING
- UPLOADING

ACK detail：

```text
+----------------+
| stopped(1)     |
+----------------+
```

## 14. CONFIG_SCAN `0x10`

```text
+----------------+----------------+------------+------------+
| scan_step(2)   | sound_speed(2) | medium(1)  | flags(1)   |
+----------------+----------------+------------+------------+
```

`flags`：

- bit0：保存为最近配置

允许状态：

- IDLE
- READY

设备行为：

1. 校验参数
2. 保存扫描配置
3. 自动执行回零
4. 回零完成后回 ACK

ACK detail：

```text
+----------------+----------------+----------------+
| zero_done(1)   | current_angle(2)| point_count(2)|
+----------------+----------------+----------------+
```

`point_count` 为本次扫描预计点数。

## 15. START_SCAN `0x11`

```text
+------------+
| start(1)   |
+------------+
```

固定 `start = 0x01`

允许状态：

- READY

ACK detail：

```text
+----------------+
| accepted(1)    |
+----------------+
```

## 16. SCAN_POINT `0x12`

```text
+----------------+----------------+----------------+----------------+
| point_index(2) | angle(2)       | distance_mm(4) | quality(2)    |
+----------------+----------------+----------------+----------------+
| valid(1)       | reserved(1)                                      |
+---------------------------------------------------------------+
```

## 17. SCAN_FINISH `0x13`

```text
+----------------+----------------+----------------+
| total_points(2)| end_reason(1)  | dropped(2)     |
+----------------+----------------+----------------+
```

`end_reason`：

- `0x00`：正常完成
- `0x01`：用户停止
- `0x02`：机械故障
- `0x03`：测量失败
- `0x04`：超时

## 18. CONFIG_DEBUG `0x20`

```text
+----------------+----------------+----------------+----------------+
| start_angle(2) | end_angle(2)   | step_angle(2)  | capture_type(1)|
+----------------+----------------+----------------+----------------+
| sample_count(2)| flags(1)                                        |
+---------------------------------------------------------------+
```

`flags`：

- bit0：单点模式
- bit1：连续模式
- bit2：上传距离结果
- bit3：上传波形

约束：

- 单点模式：`end_angle == start_angle`，`step_angle = 0`
- 连续模式：`step_angle > 0`

ACK detail：

```text
+----------------+----------------+
| accepted(1)    | normalized(1)  |
+----------------+----------------+
```

## 19. MOVE_TO_ANGLE `0x21`

```text
+----------------+
| target_angle(2)|
+----------------+
```

允许状态：

- READY

ACK detail：

```text
+----------------+----------------+----------------+----------------+
| current_angle(2)| target_angle(2)| move_dir(1)    | arrived(1)     |
+----------------+----------------+----------------+----------------+
```

`move_dir`：

- `0x00`：不动
- `0x01`：顺时针
- `0x02`：逆时针

## 20. START_SINGLE_MEASURE `0x22`

```text
+------------+
| start(1)   |
+------------+
```

允许状态：

- READY

设备行为：

1. 在当前角度执行一次采样
2. 如配置要求，上传波形
3. 上传距离结果
4. 发 WAVE_FINISH

ACK detail：

```text
+----------------+
| accepted(1)    |
+----------------+
```

## 21. WAVE_CHUNK `0x23`

```text
+----------------+----------------+----------------+----------------+
| frame_id(2)    | total_frames(2)| sample_offset(2)| sample_count(2)|
+----------------+----------------+----------------+----------------+
| angle(2)       | data_type(1)   | samples(n)                       |
+----------------+----------------+----------------------------------+
```

要求：

- 同一测量任务内，`frame_id` 从 0 递增
- 上位机必须按 `frame_id` 重组
- 如帧丢失，整次波形任务标记为不完整

## 22. WAVE_FINISH `0x24`

```text
+----------------+----------------+----------------+
| angle(2)       | total_samples(2)| result(1)     |
+----------------+----------------+----------------+
```

`result`：

- `0x00`：正常完成
- `0x01`：用户停止
- `0x02`：采样异常
- `0x03`：上传异常

## 23. DISTANCE_RESULT `0x25`

```text
+----------------+----------------+----------------+----------------+
| angle(2)       | distance_mm(4) | quality(2)     | valid(1)       |
+----------------+----------------+----------------+----------------+
```

## 24. START_DEBUG_SWEEP `0x26`

请求负载为空。

前置条件：

- 已成功执行 CONFIG_DEBUG
- debug flags 为连续模式

允许状态：

- READY

ACK detail：

```text
+----------------+
| accepted(1)    |
+----------------+
```

## 25. DEBUG_SWEEP_POINT `0x27`

```text
+----------------+----------------+----------------+----------------+
| point_index(2) | angle(2)       | distance_mm(4) | quality(2)    |
+----------------+----------------+----------------+----------------+
| valid(1)       | has_wave(1)                                      |
+---------------------------------------------------------------+
```

## 26. DEBUG_SWEEP_FINISH `0x28`

```text
+----------------+----------------+
| total_points(2)| end_reason(1)  |
+----------------+----------------+
```

## 27. STATUS_EVENT `0x7F`

```text
+----------------+----------------+----------------+----------------+
| event(1)       | status(1)      | current_angle(2)| detail(2)     |
+----------------+----------------+----------------+----------------+
```

事件码：

- `0x01`：回零开始
- `0x02`：回零完成
- `0x03`：开始转动
- `0x04`：到位
- `0x05`：开始测量
- `0x06`：测量完成
- `0x07`：波形上传开始
- `0x08`：波形上传完成
- `0xE0`：超时
- `0xE1`：堵转
- `0xE2`：采样失败
- `0xE3`：缓存溢出

## 29. 记录文件格式冻结

结果记录：

- 文件格式：CSV
- 主键：`session_id + point_index`

事件记录：

- 文件格式：JSONL
- 每行包含时间、方向、命令、序号、十六进制原始帧、解析摘要

波形记录：

- 文件格式：BIN
- 元数据索引：JSON

## 30. 实现要求

- 下位机 UART 接收必须有帧同步与 CRC 校验。
- 下位机波形上传必须支持分包。
- 上位机必须维护会话级任务状态，不允许 UI 直接裸发二进制。
- 上位机必须将正式扫描和调试模式分成两个独立任务流。

## 31. 当前冻结结论

- 正式扫描默认只传距离结果，不传全量波形。
- 调试模式允许传原始波形或包络波。
- 所有任务都必须有显式结束帧。
- 所有失败都必须给出结果码或事件码。
