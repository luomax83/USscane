# 超声声纳 360 度扫描系统主任务规划

## 1. 项目定位

本项目不是普通串口调试工具，也不是单一 STM32 测距 Demo。

目标系统是一个完整的“转台控制 + 超声采集 + 距离解算 + 波形联调 + 数据记录回放”的一体化系统，包含：

- 下位机：负责回零、转动、触发采样、距离计算、波形上传、状态管理。
- 上位机：负责模式配置、任务调度、协议通信、结果显示、波形分析、数据记录与回放。

因此，架构设计必须围绕“流程闭环”和“系统可靠性”展开，而不是围绕单个界面或单条串口指令展开。

## 2. 当前现状与目标差距

### 2.1 当前现状

- 已有 STM32 基础工程，串口与 LED 可用。
- 已有一个临时上位机骨架，适合做串口连通性验证。
- 已形成一版协议草案，覆盖扫描模式与调试模式。

### 2.2 当前差距

- 下位机仍是 LED 心跳和 ASCII 示例数据，尚未进入正式业务架构。
- 上位机仍是串口监视器，不是最终产品界面。
- 正式协议应以二进制帧为主，但现有样例仍是 ASCII。
- 尚未建立机械控制状态机、扫描任务状态机和波形分包链路。
- 尚未建立记录、回放、异常恢复、可靠性测试体系。

### 2.3 架构结论

当前代码应被视为“Bring-up 原型”，不应继续在其上直接堆业务。

正确路线是：

1. 保留现有串口样例作为联机调试工具。
2. 新建正式协议栈与业务状态机。
3. 按“先控制闭环，再结果闭环，再波形闭环”的顺序推进。

## 3. 总体架构

## 3.1 下位机架构

建议拆分为以下模块：

- `bsp`
  - GPIO、UART、DMA、TIM、ADC、编码器、驱动 IO
- `platform`
  - Tick、日志、缓存、CRC、环形队列
- `motion`
  - 回零、角度换算、方向判定、目标到位判断、停止控制
- `ultrasonic`
  - 发射触发、采样启动、原始波形缓存、包络计算、距离解算
- `protocol`
  - 二进制帧收发、命令分发、ACK/NACK、分包上传
- `app`
  - 正式扫描模式
  - 调试单点模式
  - 调试连续角度模式
  - 状态机与任务调度
- `health`
  - 超时、堵转、采样失败、缓存溢出、异常码上报

## 3.2 上位机架构

建议拆分为以下模块：

- `transport`
  - 串口连接、收发线程、超时管理、自动重连
- `protocol`
  - 帧组包、拆包、CRC、命令编解码、数据重组
- `session`
  - 扫描任务会话管理
  - 调试任务会话管理
  - 状态同步与任务取消
- `domain`
  - 扫描结果模型
  - 波形模型
  - 设备状态模型
  - 参数模板模型
- `ui`
  - 扫描页
  - 调试页
  - 日志与记录页
  - 设置页
- `storage`
  - CSV 结果保存
  - JSONL 事件与报文保存
  - 波形文件保存与回放索引
- `analysis`
  - 波形回放
  - 包络/原始波形查看
  - 结果导出

## 3.3 分层原则

- 协议层不关心 UI。
- UI 不直接拼装二进制帧。
- 距离解算与波形采集逻辑不写在串口中断里。
- 所有任务必须由状态机驱动。

## 4. 产品功能边界

## 4.1 正式扫描模式

上位机功能：

- 配置扫描步进角
- 配置介质类型
- 配置声速
- 下发配置并等待设备回零完成
- 启动扫描
- 实时显示角度序号与距离
- 显示扫描进度
- 生成极坐标或扇形结果图
- 保存扫描结果

下位机功能：

- 接收配置
- 执行回零
- 回零完成后进入 READY
- 按角度步进扫描
- 每一步完成距离测量并上传结果
- 扫描完成后上传结束帧

## 4.2 调试模式

上位机功能：

- 指定单点角度
- 连续角度范围设置
- 每次动作角度设置
- 选择上传原始波形或包络
- 发起单次测量
- 发起连续角度调试扫描
- 实时显示当前角度、方向、到位状态、距离结果、波形
- 保存波形与结果

下位机功能：

- 计算目标角度的调整方向与最短路径
- 转动到位并返回应答
- 接收开始测量命令
- 采样并上传波形分包
- 同步上传解析后的距离结果
- 连续角度调试时按范围与步进重复执行

## 5. 核心技术决策

以下决策应立即冻结，避免后续返工：

- 正式通信协议采用二进制帧。
- ASCII 输出仅保留为调试辅助，不作为正式协议。
- 角度统一单位为 `0.01°`。
- 距离统一单位为 `mm`。
- 声速统一单位为 `m/s`。
- 所有结束事件必须用“结束帧”，不能只靠单字节结束符。
- 扫描结果帧与波形数据帧分离。
- 正式扫描优先上传距离结果，不默认上传全量原始波形。
- 原始波形只在调试模式或明确开启时上传。
- 上位机采用 `PySide6 + pyserial + pyqtgraph`。

## 6. 主任务路线

项目必须按以下顺序推进：

### 阶段 A：架构冻结与模拟联调

目标：

- 冻结协议、状态机、记录格式、页面结构。
- 在没有完整硬件的情况下先打通上位机开发。

任务：

- 完成协议定稿
- 定义状态码、错误码、命令字
- 定义记录文件格式
- 编写设备模拟器
- 上位机先对接模拟器，不等待真实硬件全部完成

交付物：

- 协议文档 v1.0
- 状态机文档
- PC 端模拟器
- 上位机原型界面

### 阶段 B：下位机控制闭环

目标：

- 让设备具备“回零、到位、停止、状态上报”的稳定控制能力。

任务：

- 建立运动控制模块
- 实现回零流程
- 实现角度控制与状态机
- 实现停止命令与异常退出
- 实现异步状态上报

交付物：

- 下位机控制状态机
- 回零与到位验证记录
- 控制协议收发联调通过

验收标准：

- 连续执行 100 次回零无死锁
- 指定角度控制可重复到位
- 停止命令在运动中可打断

### 阶段 C：正式扫描模式闭环

目标：

- 跑通正式扫描的业务主链路。

任务：

- 扫描参数配置命令
- 配置成功后自动回零
- 开始扫描命令
- 每步返回扫描结果
- 扫描结束帧
- 上位机扫描页显示与保存

交付物：

- 正式扫描全流程联调完成
- 扫描结果表格与图形展示
- CSV 导出

验收标准：

- 扫描任务从配置到结束全流程无人工干预
- 结果点序号、角度、距离三者一致
- 扫描结束帧可稳定触发 UI 收尾与存盘

### 阶段 D：调试模式闭环

目标：

- 跑通单点调试与连续角度调试，支持波形上传。

任务：

- 调试参数配置
- 单点角度调整
- 到位应答
- 单次测量命令
- 原始波形/包络分包上传
- 距离结果同步上传
- 连续角度调试扫描

交付物：

- 调试模式页面
- 波形重组与显示
- 单次/连续调试任务闭环

验收标准：

- 指定角度测量可稳定返回波形和距离
- 连续角度调试结果顺序正确
- 波形分包可重组且无错帧

### 阶段 E：记录、回放与分析

目标：

- 构建项目的工程可用性，而不只是演示效果。

任务：

- 结果数据保存
- 波形数据保存
- 事件与报文日志保存
- 历史数据加载与回放
- 基础数据分析工具

交付物：

- CSV 结果文件
- JSONL 报文与事件记录
- 波形文件存储
- 回放页面

验收标准：

- 可用历史文件完整复现一次扫描过程
- 波形与距离结果可关联追溯

### 阶段 F：稳定性与系统验收

目标：

- 将系统从“能跑”提升到“可交付”。

任务：

- 异常注入测试
- 串口断连重连测试
- 长时间运行测试
- 波形大数据量压力测试
- 错误码覆盖测试
- 参数边界测试

交付物：

- 测试报告
- 问题清单与修复记录
- 发布基线版本

验收标准：

- 长时间运行不死锁
- 异常情况下状态机可恢复或可控退出
- 关键命令和关键事件均可追踪

## 9. 风险清单与控制策略

### 风险 1：机械控制和测量耦合过深

风险：

- 转动逻辑和测量逻辑写在一起，后期难维护。

对策：

- 强制分层，运动控制只负责“到位”，测量模块只在 READY 状态触发。

### 风险 2：波形数据量导致串口成为瓶颈

风险：

- 原始波形上传过大，阻塞控制链路。

对策：

- 正式扫描默认只上传结果。
- 波形只在调试模式上传。
- 分包、限速、必要时提高波特率。

### 风险 3：状态机混乱

风险：

- 回零、运动、测量、上传之间的切换不一致，造成死锁。

对策：

- 所有命令必须定义允许状态与禁止状态。
- 任何动作完成都必须回到明确状态。

### 风险 4：上位机先做成“串口脚本集合”

风险：

- 代码很快失控，后期难扩展。

对策：

- 尽快完成 transport、protocol、session、ui 四层分离。

## 10. 验收口径

项目最终验收不能只看“有界面”和“能收到数据”，必须满足：

- 正式扫描模式全流程闭环
- 调试模式单点和连续扫描闭环
- 波形与距离结果可关联
- 日志可保存并回放
- 故障和异常可定位
- 系统可长时间运行

## 11. 立即生效的项目规则

- 现有 ASCII 示例协议不再继续扩展业务功能，只保留为 bring-up 工具。
- 正式功能统一围绕二进制协议实现。
- 先打通扫描模式，再做调试波形深水区。
- 任何新增命令都必须先定义状态约束和错误码。
- 任何页面功能都必须能映射到协议命令和设备状态。
